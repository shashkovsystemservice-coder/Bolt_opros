# Модуль Контактов — Документация

## ⚠️ Терминология БД (важно!)

### Таблица `companies` ≠ "компании клиентов"
Эта таблица хранит **администраторов** Survey Pro (тех, кто создаёт опросы).

### Таблица `participants` = контакты/респонденты
Это люди, которым администраторы отправляют опросы для заполнения.

## Поля и их значение

### В `participants`:
- **`company_id`** — ID администратора, который добавил контакт (FK → companies.id)
- **`company_name`** — название компании, где работает сам контакт

## Пример

Администратор "ООО Консалтинг" добавил контакт "Иванов Пётр из Сбербанка":

```sql
-- Таблица companies (администраторы)
-- id: "abc123..."
-- email: "admin@consulting.com"
-- name: "ООО Консалтинг"

-- Таблица participants (контакты)
-- id: "xyz789..."
-- company_id: "abc123..."        ← ID админа "ООО Консалтинг"
-- first_name: "Иванов"
-- email: "ivanov@sberbank.ru"
-- company_name: "Сбербанк"       ← Где работает Иванов
```

## Права доступа (RLS)
- **Обычный админ**: видит только свои контакты (WHERE company_id = auth.uid())
- **Супер-админ**: видит все контакты всех администраторов
